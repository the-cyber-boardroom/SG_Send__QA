Bug tests follow the pattern: a test that expects the correct behaviour is written first, it fails (documenting the bug), and the test is marked xfail until the fix ships. When the fix ships, xfail is removed and the test goes green.

How Bugs Are Tracked

  1. Detection — An automated test fails, or a manual observation raises a concern. A screenshot captures the broken state.
  2. Bug test written — A test is written that asserts the correct behaviour. It fails intentionally. The test is marked @pytest.mark.xfail(reason="bug #NNN — description").
  3. Tracked here — The bug is listed on this page with a link to the test file, the GitHub issue, and the screenshot evidence.
  4. Fix shipped — When SG/Send fixes the bug and the test goes green, xfail is removed. The bug moves to "Recently Fixed".

Reporting a Bug

Open an issue on the SG/Send QA GitHub repository with steps to reproduce and, if possible, a screenshot.
