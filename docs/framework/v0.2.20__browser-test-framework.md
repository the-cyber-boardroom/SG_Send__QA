# SG/Send QA — Browser Test Framework Reference

**Version:** v0.2.20
**Date:** 2026-03-25
**Package:** `sg_send_qa/browser/`

---

## Overview

The browser test framework is a thin layer on top of `osbot_playwright` that provides:

1. **Lifecycle management** — start/stop the local API server, UI server, and browser as one unit
2. **Shadow DOM-aware page primitives** — named methods for every SG/Send UI interaction, hiding all selectors
3. **Stable port persistence** — same ports across debug sessions → localStorage survives
4. **State capture** — typed schemas for each page's data model, serialisable to JSON

The framework is used by both integration tests (`tests/integration/browser/`) and (progressively) by v030 QA acceptance tests (`tests/qa/v030/`).

## Class Reference

### `SG_Send__Browser__Test_Harness`

**File:** `sg_send_qa/browser/SG_Send__Browser__Test_Harness.py`

The top-level lifecycle manager. One instance per test class. Manages:
- In-memory FastAPI server (SG/Send User Lambda, random or persisted port)
- Static UI server (built from `sgraph_ai_app_send__ui__user` package, stable or cached)
- Headless/headed Chromium browser session
- Access token injection via localStorage
- State persistence to disk (debug mode) or fresh start (CI mode)

**Constructor & Fluent Setup:**
```python
harness = SG_Send__Browser__Test_Harness()
harness = harness.headless(True)   # True = CI mode; False = debug/headed
harness = harness.setup()          # start everything
```

Or chained:
```python
harness = SG_Send__Browser__Test_Harness().headless(True).setup()
```

**Key Methods:**

| Method | Returns | Description |
|---|---|---|
| `headless(bool)` | `self` | Set headless/headed mode (must call before `setup()`) |
| `setup()` | `self` | Start API server, build + serve UI, launch browser |
| `teardown()` | `self` | Stop all services, close browser |
| `set_access_token()` | `str` | Inject access token into localStorage; returns token |
| `transfer_helper()` | `QA_Transfer_Helper` | Pre-wired transfer helper for creating test uploads |

**Context Manager (for access token injection):**
```python
with harness as _:
    access_token = _.set_access_token()
    ui_server    = _.ui_server
```

**Attributes:**

| Attribute | Type | Description |
|---|---|---|
| `config` | `Schema__Browser_Test_Config` | headless, port config |
| `persistence` | `Harness_State__Persistence` | disk-based state |
| `api_server` | `Fast_API_Server` | the in-memory API server |
| `ui_server` | `Temp_Web_Server` | the static file server |
| `sg_send` | `SG_Send__Browser__Pages` | page interaction primitives |
| `test_objs` | `Send__User_Lambda__Test_Objs` | access_token, server_url |
| `ui_serve_dir` | `str` | absolute path to built UI files |

**Debug Mode vs CI Mode:**

| Aspect | CI (`headless=True`) | Debug (`headless=False`) |
|---|---|---|
| Ports | Random each run | Persisted to disk |
| UI build | Fresh each run | Cached by UI version |
| Browser | Headless | Headed (visible) |
| localStorage | Cleared each run | Survives between runs |
| State file | Not written | `~/.tmp/sg_send_qa_harness/harness_state.json` |

### `SG_Send__Browser__Pages`

**File:** `sg_send_qa/browser/SG_Send__Browser__Pages.py`

The page primitives layer. All SG/Send UI interactions go through here. No test code ever holds a raw Playwright `Page` or uses raw locators directly.

**Constructor:**
```python
# Normally obtained from the harness:
sg_send = harness.sg_send

# Or standalone (for advanced use):
sg_send = SG_Send__Browser__Pages(headless=True, target_port=10062)
```

**Context Manager (recommended for test methods):**
```python
with sg_send as _:
    _.page__root()
    _.upload__set_file("test.txt", b"hello")
    # ...
```

| Method | Returns | Description |
|---|---|---|

#### Wait Methods

| Method | Description |
|---|---|
| `wait(ms)` | Explicit timeout wait. Use sparingly — only when no event-based alternative exists. |
| `wait_for_page_ready(timeout)` | Wait for `body[data-ready]` attribute |
| `wait_for_component(tag_name, timeout)` | Wait for a Web Component to be attached |
| `wait_for_upload_state(state, timeout)` | Poll `send-upload._state` until it matches |

**Upload states:** `'idle'` → `'file-ready'` → `'choosing-delivery'` → `'choosing-share'` → `'confirming'` → `'uploading'` → `'complete'`

#### localStorage / Storage Methods

| Method | Description |
|---|---|
| `storage__set_token(token)` | Write access token to localStorage |
| `storage__get_token()` | Read access token from localStorage via qa-setup page |
| `storage__clear()` | Wipe all localStorage |
| `storage__dump()` | Return all localStorage as dict |

#### Access Gate Methods

The access gate (`<send-access-gate>`) renders in light DOM via `innerHTML`.

| Method | Description |
|---|---|
| `access_gate__enter_token(token)` | Wait for input, fill token |
| `access_gate__submit()` | Click Go button (`#access-token-submit`), wait 500ms |
| `access_gate__enter_and_submit(token)` | Combined fill + submit |

**Shadow DOM note:** The Go button is the **second** button in the gate. The first is the eye icon (`#toggle-token-vis`). Always use `#access-token-submit` explicitly — never `button.first`.

#### Upload Wizard Methods

The upload wizard (`<send-upload>`) is a light DOM orchestrator containing Shadow DOM sub-components:

```
<send-upload>                    light DOM (state machine)
  └─ <upload-step-select>        shadow DOM — #file-input, #drop-zone
  └─ <upload-step-delivery>      shadow DOM
  └─ <upload-step-share>         shadow DOM — .share-card[data-mode]
  └─ <upload-step-confirm>       shadow DOM
  └─ <upload-step-progress>      shadow DOM
  └─ <upload-step-done>          shadow DOM — #combined-link, #simple-token
#upload-next-btn                 light DOM (rendered by orchestrator)
```

| Method | Shadow DOM? | Description |
|---|---|---|
| `upload__set_file(filename, bytes, mime)` | ✅ Pierces | Set file via `upload-step-select #file-input` |
| `upload__click_next()` | No | Click `#upload-next-btn`, wait 500ms |
| `upload__select_share_mode(mode)` | ✅ Pierces | Click `upload-step-share [data-mode="X"]`, wait 500ms |
| `upload__wait_for_complete(timeout)` | — | Poll until `send-upload._state === 'complete'` |
| `upload__get_combined_link()` | ✅ Pierces | Read `upload-step-done #combined-link` |
| `upload__get_friendly_token()` | ✅ JS eval | Shadow DOM JS evaluation for `#simple-token` |
| `upload__get_full_link()` | ✅ Pierces | Read `upload-step-done #full-link` |
| `upload__get_link_only()` | ✅ Pierces | Read `upload-step-done #link-only` |
| `upload__get_decryption_key()` | ✅ Pierces | Read `upload-step-done #decryption-key` |

**Shadow DOM piercing pattern:**
```python
# CSS descendant combinator pierces open shadow roots:
self.raw_page().locator("upload-step-select #file-input")
#                        └─ shadow host ─┘ └─ element in shadow root ─┘
```

### `Harness_State__Persistence`

**File:** `sg_send_qa/browser/Harness_State__Persistence.py`

Saves and restores harness port configuration between runs. Enables debug mode stability.

**State file location:** `~/.tmp/sg_send_qa_harness/harness_state.json`

**Persisted values:**

| Field | Description |
|---|---|
| `api_port` | Port of the in-memory API server |
| `ui_port` | Port of the static UI server |
| `ui_build_folder` | Absolute path to cached UI build |
| `ui_version` | UI package version (cache key) |
| `access_token` | Last-used access token |
| `chrome_port` | CDP port for the browser process |

**How it works:**
1. On `setup()`: load saved state → check if ports are still free → reuse if yes, allocate fresh if no
2. On `teardown()`: save current state to disk
3. CI mode: `config.headless=True` → state file not written

## Test Patterns

### Pattern 1: Integration Test (ordered, shared state)

```python
from unittest import TestCase
from sg_send_qa.browser.SG_Send__Browser__Test_Harness import SG_Send__Browser__Test_Harness

SAMPLE_CONTENT  = b"Test content for SG/Send QA"
SAMPLE_FILENAME = "qa-test.txt"

class test_Upload_And_Download_Workflow(TestCase):

    @classmethod
    def setUpClass(cls):
        cls.harness = SG_Send__Browser__Test_Harness().headless(True).setup()
        cls.sg_send = cls.harness.sg_send
        with cls.harness as _:
            cls.access_token = _.set_access_token()

    @classmethod
    def tearDownClass(cls):
        cls.harness.teardown()

    def test__01__upload_file(self):
        with self.sg_send as _:
            _.page__root()
            _.upload__set_file(SAMPLE_FILENAME, SAMPLE_CONTENT)
            _.upload__click_next()                        # → choosing-share
            _.upload__select_share_mode("combined")       # → confirming
            _.upload__click_next()                        # → uploading
            _.upload__wait_for_complete()
            assert _.upload_state() == 'complete'
            # Store link for test_02 via localStorage
            link = _.upload__get_combined_link()
            _.js_evaluate(f"localStorage.setItem('qa-last-link', '{link}')")

    def test__02__download_via_combined_link(self):
        link = self.sg_send.js_evaluate("localStorage.getItem('qa-last-link')")
        assert link and '#' in link
        hash_part = link.split('#')[1]
        tid, key  = hash_part.split('/', 1)
        with self.sg_send as _:
            _.page__browse_with_hash(tid, key)
            _.wait(3000)        # allow decrypt + render
            text = _.visible_text()
            assert SAMPLE_FILENAME in text
```

### Pattern 2: v030 QA Acceptance Test (independent, screenshot-producing)

```python
import pytest
pytestmark = pytest.mark.p0

class TestSingleFileUpload:

    def test_single_file_upload_flow(self, sg_send, send_server, screenshots):
        """Upload a single text file and verify the download link is shown."""
        with sg_send as _:
            _.page__root()
            screenshots.capture(_.raw_page(), "01_upload_page", "Upload page loaded")
            _.upload__set_file("test.txt", b"Hello from QA")
            screenshots.capture(_.raw_page(), "02_file_selected", "File selected")
            _.upload__click_next()
            _.upload__select_share_mode("combined")
            screenshots.capture(_.raw_page(), "03_mode_selected", "Combined mode")
            _.upload__click_next()
            _.upload__wait_for_complete()
            screenshots.capture(_.raw_page(), "04_done", "Upload complete")
            link = _.upload__get_combined_link()
        assert '#' in link
```

### Pattern 3: Unit Test for Framework Class

```python
from unittest import TestCase
from sg_send_qa.browser.Schema__Browser_Test_Config import Schema__Browser_Test_Config

class test_Schema__Browser_Test_Config(TestCase):

    def test_defaults(self):
        config = Schema__Browser_Test_Config()
        assert config.headless      is True
        assert config.capture_stderr is False
        assert config.api_port      == 0
        assert config.ui_port       == 0

    def test_json_round_trip(self):
        import json
        config = Schema__Browser_Test_Config(headless=False, ui_port=10062)
        data   = config.__dict__
        config2 = Schema__Browser_Test_Config(**json.loads(json.dumps(data)))
        assert config2.headless == False
        assert config2.ui_port  == 10062
```

## Shadow DOM Piercing Reference

SG/Send v0.3.0 uses Web Components with open shadow roots. Playwright CSS selectors pierce open shadow roots automatically using the **descendant combinator** (space between shadow host and inner element):

```python
# ✅ Pierces open shadow root:
page.locator("upload-step-select #file-input")
page.locator("upload-step-share [data-mode='combined']")
page.locator("upload-step-done #combined-link")

# ✅ JS eval always works (explicit shadow traversal):
page.evaluate("document.querySelector('upload-step-done').shadowRoot.querySelector('#simple-token').innerText")

# ❌ Does NOT pierce (v030 legacy pattern — only works if element is in light DOM):
page.locator("#file-input")
```

**Rule:** Any element that lives inside a Web Component's shadow root must be accessed via CSS piercing (`"component-tag element-selector"`) or JS evaluation. Never bare `#id` selectors for shadow DOM content.

## Known Limitations

1. **`wait(ms)` calls** — some methods use explicit ms waits where event-based waits are not yet implemented. These are documented with comments. Future work: replace with `wait_for_upload_state()` equivalents.

2. **`storage__set_token()` XSS** — token is interpolated directly into a JS string. Acceptable for QA (tokens are internal GUIDs) but must not be used with external/user-supplied input.

3. **Single page per harness** — the harness uses one browser page shared across all tests in a class. Tests that navigate away break subsequent tests. Design tests as ordered sequences within a class, or call `page__root()` at the start of each test.

4. **`_current_ui_version()` cache key** — currently reads from `sg_send_qa/version` instead of the UI package version. A QA version bump incorrectly invalidates the UI cache. Fix: read from `sgraph_ai_app_send__ui__user.__version__`.

5. **Download page shadow DOM** — `download__enter_key()` uses `#key-input` which may live in shadow DOM. If tests fail on this selector, apply CSS piercing: `"send-download #key-input"`.
