## What This Brief Covers

This brief specifies the complete redesign of the QA screenshot pipeline. The core
architectural insight — that **screenshots should only be written to S3 when the UI
actually changes** — has cascading implications for the S3 folder structure, the capture
logic, the manifest, and how the QA site resolves image URLs. All of these are explained
in full detail below so the implementing Claude Code session can build this correctly
without ambiguity.

Read this document in order. Do not skip sections.

## Part 1 — The Core Insight (Read This First)

### The problem with the v1 approach

The first draft of this brief used `run_id` (an ISO timestamp) as the S3 folder key:

```
screenshots/sgraph-ai/2026-03-10T14:22:00Z/pt-pt/pricing/01_hero.png
screenshots/sgraph-ai/2026-03-10T16:00:00Z/pt-pt/pricing/01_hero.png   ← identical bytes
screenshots/sgraph-ai/2026-03-11T09:15:00Z/pt-pt/pricing/01_hero.png   ← identical bytes
```

This is wrong. Every CI run would write a full set of screenshots — 80+ PNGs per run — even
when nothing in the target site changed. Storage grows without bound. More importantly, the
data is meaningless: you cannot tell from the S3 structure whether anything changed between
runs.

### The right model: version-keyed, write-on-change only

The insight is: **a screenshot only has new information when the UI it captures has changed.**
If `pt-pt/pricing/01_hero.png` was identical in 50 consecutive test runs, there is exactly
one useful copy of that image. The version of the target site when it was *first captured
in that state* is the meaningful key — not the timestamp of any individual test run.

The S3 structure becomes:

```
screenshots/sgraph-ai/
  v0.10.5/
    pt-pt/pricing/01_hero.png      ← first captured at v0.10.5
  v0.12.3/
    pt-pt/pricing/01_hero.png      ← UI changed in v0.12.3, new copy written
    en-gb/home/01_full_page.png    ← home page also changed in v0.12.3
  v0.13.8/
    de-de/pricing/01_hero.png      ← only German pricing changed in v0.13.8
```

Note what is **absent**: `v0.13.8/pt-pt/pricing/01_hero.png` does not exist, because
that image has not changed since v0.12.3. `v0.13.8/en-gb/home/01_full_page.png` does
not exist either, for the same reason.

This is **immutable storage**. Once written, a versioned screenshot is never overwritten.
It is a permanent record of what that page looked like at that version of the target.

### The manifest: a pointer map to the latest version of each screenshot

Because a screenshot may not exist under the current target version folder, the system
needs a way to answer: *"What is the current screenshot for `pt-pt/pricing/01_hero`?"*

The answer lives in a manifest file:

```
screenshots/sgraph-ai/manifest.json
```

The manifest maps every `(locale, page, step)` triple to the target version that contains
the most recent copy of that screenshot:

```json
{
  "schema_version": 1,
  "target":         "sgraph-ai",
  "target_version": "v0.13.8",
  "updated_at":     "2026-03-10T14:22:00Z",
  "entries": {
    "en-gb/home/01_full_page":            "v0.12.3",
    "en-gb/home/02_above_fold":           "v0.12.3",
    "en-gb/pricing/01_full_page":         "v0.13.8",
    "en-gb/pricing/02_above_fold":        "v0.13.8",
    "pt-pt/home/01_full_page":            "v0.12.3",
    "pt-pt/pricing/01_full_page":         "v0.12.3",
    "de-de/pricing/01_full_page":         "v0.13.8",
    "fr-fr/home/01_full_page":            "v0.10.5"
  }
}
```

To resolve the S3 URL for any screenshot:

```python
def resolve_screenshot_url(manifest, bucket_base_url, target, locale, page, step):
    entry_key    = f"{locale}/{page}/{step}"
    ifd_version  = manifest["entries"][entry_key]
    s3_key       = f"websites/sgraph-qa/screenshots/{target}/{ifd_version}/{entry_key}.png"
    return f"{bucket_base_url}/{s3_key}"
    # → https://745506449035--static-sgraph-ai--eu-west-2.s3.eu-west-2.amazonaws.com/
    #     websites/sgraph-qa/screenshots/sgraph-ai/v0.12.3/pt-pt/pricing/01_full_page.png
```

The QA site HTML never hardcodes S3 paths. It always resolves through the manifest.

### What this enables

- **Visual changelog for free.** Diff two consecutive manifests to see exactly which pages
  changed visually between any two versions of the target site. No additional tooling needed.
- **Efficient storage.** Screenshots that never change are stored exactly once.
- **LLM analysis is also version-keyed.** The `.json` analysis file for each screenshot
  lives at the same path as the `.png` — `{ifd_version}/{locale}/{page}/{step}.json`.
  It is only written when the screenshot is written, so analysis results are also immutable
  and permanently associated with a specific visual state.
- **Instant "what changed?" queries.** `manifest["entries"]` tells you, for every single
  screenshot, exactly when it last changed. No S3 listing required.

### Critical separation

The **QA site tree** (`dev/`, `main/`, `prod/`) and the **screenshots tree** are siblings
under `websites/sgraph-qa/`. They are entirely independent:

- The QA site version controls what HTML is deployed. It bumps on every push to dev/main.
- The screenshot version is the target site's version. It only creates new folders when
  the target UI changes.
- Neither tree references the other by version. They are linked only at runtime through
  the manifest URL, which the QA site fetches when rendering a use-case page.

### S3 key helpers (Python)

Put these in `sg_send_qa/utils/s3_keys.py`:

```python
"""S3 key construction for QA screenshot storage.

All paths are relative to the bucket root.
Bucket: 745506449035--static-sgraph-ai--eu-west-2
"""

BUCKET      = "745506449035--static-sgraph-ai--eu-west-2"
REGION      = "eu-west-2"
BUCKET_URL  = f"https://{BUCKET}.s3.{REGION}.amazonaws.com"
SCREENSHOTS = "websites/sgraph-qa/screenshots"


def screenshot_key(target: str, target_version: str, locale: str,
                   page: str, step_name: str) -> str:
    """
    S3 key for a single screenshot PNG.

    Example:
        screenshot_key("sgraph-ai", "v0.13.8", "pt-pt", "pricing", "01_full_page")
        → "websites/sgraph-qa/screenshots/sgraph-ai/v0.13.8/pt-pt/pricing/01_full_page.png"
    """
    return f"{SCREENSHOTS}/{target}/{target_version}/{locale}/{page}/{step_name}.png"


def analysis_key(target: str, target_version: str, locale: str,
                 page: str, step_name: str) -> str:
    """
    S3 key for the LLM analysis JSON alongside a screenshot.
    Same path as screenshot_key but with .json extension.
    """
    return f"{SCREENSHOTS}/{target}/{target_version}/{locale}/{page}/{step_name}.json"


def manifest_key(target: str) -> str:
    """
    S3 key for the manifest file for a given target.

    Example:
        manifest_key("sgraph-ai")
        → "websites/sgraph-qa/screenshots/sgraph-ai/manifest.json"
    """
    return f"{SCREENSHOTS}/{target}/manifest.json"


def screenshot_url(target: str, target_version: str, locale: str,
                   page: str, step_name: str) -> str:
    """Public HTTPS URL for a screenshot."""
    return f"{BUCKET_URL}/{screenshot_key(target, target_version, locale, page, step_name)}"


def manifest_url(target: str) -> str:
    """Public HTTPS URL for a target's manifest."""
    return f"{BUCKET_URL}/{manifest_key(target)}"
```

## Part 3 — The Manifest in Full Detail

### File location

```
websites/sgraph-qa/screenshots/{target}/manifest.json
```

One manifest per target. For Phase 1 there is one target (`sgraph-ai`), so one manifest:

```
websites/sgraph-qa/screenshots/sgraph-ai/manifest.json
```

### Schema

```json
{
  "schema_version":  1,
  "target":          "sgraph-ai",
  "target_version":  "v0.13.8",
  "updated_at":      "2026-03-10T14:22:00Z",
  "qa_version":      "v0.3.1",
  "entries": {
    "{locale}/{page}/{step}": "{target_version_when_last_changed}"
  }
}
```

Field definitions:

| Field | Type | Meaning |
|-------|------|---------|
| `schema_version` | int | Always 1 for now. Increment if the schema changes. |
| `target` | str | Identifier for the target site. Matches the folder name. |
| `target_version` | str | The target site version at the time of the last test run that updated this manifest. |
| `updated_at` | ISO str | Timestamp of when this manifest was last written to S3. |
| `qa_version` | str | The QA project version (`sg_send_qa/version`) at time of last update. Useful for tracing which QA code produced a given manifest state. |
| `entries` | dict | Keys are `"{locale}/{page}/{step}"`. Values are the `target_version` folder containing the latest copy of that screenshot. |

### Entry key format

```
"{locale}/{page}/{step}"
```

- `locale` — the locale slug, e.g. `en-gb`, `pt-pt`, `de-de`, `fr-fr`
- `page` — the page name from `test-config.json`, e.g. `home`, `pricing`, `pricing-cloud`
- `step` — the step name without extension, e.g. `01_full_page`, `02_above_fold`, `03_nav`

Examples:
```
"en-gb/home/01_full_page"
"pt-pt/pricing/01_full_page"
"de-de/pricing-cloud/02_above_fold"
"fr-fr/agents-ambassador/01_full_page"
```

### What "latest version of that screenshot" means

The value in `entries` is the `target_version` folder where the **current visual state**
of that screenshot lives. When the test runner captures a new screenshot and compares it
to the existing one:

- **If pixel diff > threshold:** the screenshot has changed. Write it to the new
  `target_version` folder. Update `entries[key]` to point to `target_version`.
- **If pixel diff ≤ threshold:** the screenshot has not meaningfully changed. Do NOT write
  to S3. Leave `entries[key]` pointing to the previous version. That older copy IS the
  correct current screenshot.

This means `entries[key]` always points to the version of the target that **introduced**
the current visual state of that page/locale/step, not the version that last tested it.

### Reading the manifest as a visual changelog

The manifest makes the following queries trivially answerable without any S3 listing:

```python
# "What changed visually in v0.13.8?"
changed_in_v0138 = [k for k, v in manifest["entries"].items() if v == "v0.13.8"]

# "When did de-de/pricing last change?"
last_changed = manifest["entries"]["de-de/pricing/01_full_page"]

# "Has en-gb/home changed since v0.12.3?"
current = manifest["entries"]["en-gb/home/01_full_page"]
changed = current != "v0.12.3"

# "Which pages still show v0.10.5 state?" (potential staleness concern)
still_old = [k for k, v in manifest["entries"].items() if v == "v0.10.5"]
```

## Part 4 — The Capture Algorithm (Step by Step)

This is the heart of the implementation. The developer implementing this must understand
every step. Do not approximate this logic.

### Where this lives

`sg_send_qa/ci/screenshot_pipeline.py` — a new module, replacing/extending
`diff_screenshots.py`. The existing `diff_screenshots.py` operates on git; this new module
operates on S3. Both use the same `pixel_diff_ratio()` function from Pillow.

### Inputs

```python
target         : str    # e.g. "sgraph-ai"
target_version : str    # e.g. "v0.13.8"  — read from target at test time (see §6)
locale         : str    # e.g. "pt-pt"
page           : str    # e.g. "pricing"
step_name      : str    # e.g. "01_full_page"
new_png_bytes  : bytes  # freshly captured screenshot
threshold      : float  # default 0.01 (1%) — from test-config.json
```

### Step-by-step algorithm

```
1. LOAD MANIFEST
   ─────────────
   Read manifest.json from S3:
     key = manifest_key(target)  →  "websites/sgraph-qa/screenshots/sgraph-ai/manifest.json"

   If the manifest does not exist (first-ever run for this target):
     Create an empty manifest in memory:
       { "schema_version": 1, "target": target, "entries": {} }
     Proceed to step 3 (no existing screenshot to compare against).

2. LOOK UP EXISTING SCREENSHOT
   ────────────────────────────
   entry_key = f"{locale}/{page}/{step_name}"   # e.g. "pt-pt/pricing/01_full_page"

   If entry_key is NOT in manifest["entries"]:
     This screenshot has never been captured before.
     Proceed to step 4 (write unconditionally).

   existing_version = manifest["entries"][entry_key]
   # e.g. "v0.12.3"

   existing_s3_key = screenshot_key(target, existing_version, locale, page, step_name)
   # → "websites/sgraph-qa/screenshots/sgraph-ai/v0.12.3/pt-pt/pricing/01_full_page.png"

   Fetch existing_png_bytes from S3 using existing_s3_key.

   If S3 fetch fails (key not found — should not happen if manifest is consistent):
     Log a warning: "Manifest pointed to {existing_s3_key} but key not found in S3."
     Proceed to step 4 (treat as new screenshot).

3. COMPARE
   ────────
   ratio = pixel_diff_ratio(
       Image.open(BytesIO(existing_png_bytes)),
       Image.open(BytesIO(new_png_bytes))
   )

   If ratio <= threshold:
     Log: "UNCHANGED  {entry_key}  (diff={ratio:.4%} <= {threshold:.1%})"
     Log: "  → still at {existing_version}"
     DO NOT write to S3.
     DO NOT update manifest["entries"][entry_key].
     Return result: { "action": "unchanged", "version": existing_version, "diff": ratio }

   If ratio > threshold:
     Log: "CHANGED    {entry_key}  (diff={ratio:.4%} > {threshold:.1%})"
     Log: "  → was {existing_version}, now {target_version}"
     Proceed to step 4.

4. WRITE TO S3
   ────────────
   new_s3_key = screenshot_key(target, target_version, locale, page, step_name)
   # → "websites/sgraph-qa/screenshots/sgraph-ai/v0.13.8/pt-pt/pricing/01_full_page.png"

   Upload new_png_bytes to S3:
     - ContentType:    "image/png"
     - CacheControl:   "public, max-age=31536000"  ← 1 year: immutable, never changes
     - ACL:            public-read (screenshots must be publicly accessible by QA site)

   Also write the LLM analysis stub:
     analysis_s3_key = analysis_key(target, target_version, locale, page, step_name)
     stub = {
       "schema_version": 1,
       "target":         target,
       "target_version": target_version,
       "locale":         locale,
       "page":           page,
       "step":           step_name,
       "verdict":        None,      ← null until LLM layer is wired
       "analysis":       None,
       "captured_at":    datetime.utcnow().isoformat() + "Z"
     }
     Upload JSON stub to analysis_s3_key.
     - ContentType: "application/json"
     - CacheControl: "public, max-age=300"  ← short cache: will be updated when LLM runs

5. UPDATE MANIFEST IN MEMORY
   ──────────────────────────
   manifest["entries"][entry_key] = target_version

   NOTE: Do NOT write the manifest to S3 yet. The manifest is written ONCE at the
   end of the full test run, after all screenshots have been processed. Writing it
   per-screenshot would create race conditions and unnecessary S3 writes.

   Return result: { "action": "written", "version": target_version, "diff": ratio }
```

### After all screenshots are processed: write the manifest

```
6. WRITE MANIFEST
   ───────────────
   Update top-level fields:
     manifest["target_version"] = target_version
     manifest["updated_at"]     = datetime.utcnow().isoformat() + "Z"
     manifest["qa_version"]     = read_file("sg_send_qa/version").strip()

   Upload to S3:
     key = manifest_key(target)
     # → "websites/sgraph-qa/screenshots/sgraph-ai/manifest.json"
     ContentType:  "application/json"
     CacheControl: "public, max-age=60"   ← short cache: manifest changes on every run
                                             that has any changes

   Log summary:
     "Manifest written: {unchanged} unchanged, {written} new/changed"
     "  Target: {target} at {target_version}"
     "  Manifest: {manifest_url(target)}"
```

### What happens when nothing changed

If every single screenshot in the run is unchanged (ratio ≤ threshold for all), the manifest
top-level fields (`target_version`, `updated_at`, `qa_version`) are still updated and written
to S3, because we want the manifest to record "this version was tested and confirmed identical."
No new screenshot folders are created.

### Updated screenshots fixture

```python
@pytest.fixture
def screenshots(request, shot_context):
    """Screenshot capture fixture with S3 upload and manifest tracking."""
    # Derive page name from test file path, e.g.:
    #   tests/integration/sgraph_ai/test_pricing.py → "pricing"
    page_name  = Path(request.fspath).stem.replace("test_", "")
    locale     = shot_context["locale"]
    target     = shot_context["target"]

    # Local disk path (fallback + debug)
    local_dir  = Path("sg_send_qa__site/pages/use-cases") / f"{target}__{locale}__{page_name}" / "screenshots"
    local_dir.mkdir(parents=True, exist_ok=True)

    captured  = []
    s3_enabled = bool(os.environ.get("AWS_ACCESS_KEY_ID"))

    class ScreenshotCapture:
        def capture(self, page, step_name, description=""):
            # 1. Capture to local disk
            local_path = local_dir / f"{step_name}.png"
            _cdp_screenshot(page, str(local_path))
            png_bytes = local_path.read_bytes()

            # 2. Upload to S3 if credentials available
            if s3_enabled:
                from sg_send_qa.ci.screenshot_pipeline import process_screenshot
                result = process_screenshot(
                    target         = target,
                    target_version = shot_context["target_version"],
                    locale         = locale,
                    page           = page_name,
                    step_name      = step_name,
                    new_png_bytes  = png_bytes,
                )
                captured.append({
                    "step_name":   step_name,
                    "description": description,
                    "action":      result["action"],    # "written" or "unchanged"
                    "version":     result["version"],   # which version contains this screenshot
                    "diff":        result.get("diff"),
                })
            else:
                captured.append({
                    "step_name":   step_name,
                    "description": description,
                    "action":      "local_only",
                    "version":     "local",
                    "diff":        None,
                })

        @property
        def all(self):
            return captured

        def save_metadata(self):
            meta = {
                "target":         target,
                "target_version": shot_context["target_version"],
                "locale":         locale,
                "page":           page_name,
                "screenshots":    captured,
                "test_doc":       request.node.obj.__doc__ or "",
            }
            (local_dir / "_metadata.json").write_text(json.dumps(meta, indent=2))

    capture = ScreenshotCapture()
    yield capture
    capture.save_metadata()
```

### Manifest finalisation (session-scoped)

The manifest must be written once after ALL tests complete, not once per test. Use a
session-scoped fixture with `autouse=True`:

```python
@pytest.fixture(scope="session", autouse=True)
def finalise_manifest(shot_context):
    """Write the manifest to S3 after all tests have run."""
    yield   # ← all tests run here

    if not os.environ.get("AWS_ACCESS_KEY_ID"):
        return   # local dev — no manifest to write

    from sg_send_qa.ci.screenshot_pipeline import write_manifest
    write_manifest(target=shot_context["target"], target_version=shot_context["target_version"])
```

The `write_manifest()` function reads the in-memory manifest that was built up during
the session (see §4, step 6) and uploads it to S3.

**Important:** `screenshot_pipeline.py` must hold the in-memory manifest state as a
module-level variable (or a simple singleton), so that all calls to `process_screenshot()`
within a session accumulate into the same manifest object, which `write_manifest()` then
persists.

## Part 6 — Resolving the Target Version

The `target_version` (e.g. `v0.13.8`) is the IFD version of the **target site at the
time the test runs**. It must be read from the live target, not from the QA repo.

### How to read it for sgraph.ai

Every deployed SGraph site exposes its IFD version as a plain text file at `{base_url}/version`.
This is the same convention used throughout the ecosystem (`sg_send_qa/version`,
`sgraph_ai_app_send/version`, etc.) — one line, content like `v0.13.8`.

**Implement as a Python function** in `sg_send_qa/utils/target_version.py`:

```python
import urllib.request

def get_target_version(target: str, base_url: str) -> str:
    """
    Fetch the current version of the target site from its /version file.

    Follows the SGraph convention: every deployed site exposes its IFD
    version as plain text at {base_url}/version.

    Returns "unknown" if the file is unreachable or malformed.
    """
    try:
        with urllib.request.urlopen(f"{base_url}/version", timeout=10) as resp:
            return resp.read().decode("utf-8").strip()
    except Exception as e:
        print(f"  WARNING: Could not read target version from {base_url}/version: {e}")
        return "unknown"
```

No HTML parsing, no regex, no fragility. `urllib.request` is stdlib — no new dependency.

### ⚠️ DevOps prerequisite: Cache-Control on the `version` file

The `version` file has no extension, so `deploy_static_site.py` will not automatically
assign it a content type or cache policy. It must be handled explicitly, otherwise
CloudFront may serve a stale version and CI will tag screenshots against the wrong

## Part 7 — S3 Operations: Using the AWS CLI (Consistent with deploy_static_site.py)

The existing `deploy_static_site.py` uses `aws s3 sync` via `subprocess`. The QA project
must follow the same pattern — **no boto3, no AWS SDK**. All S3 operations go through the
`aws` CLI invoked via `subprocess`.

Note: `boto3` is not in `requirements.txt` and must not be added. The `aws` CLI is
available in the GitHub Actions runner and in any environment with credentials configured.

### S3 operation helpers

Put in `sg_send_qa/utils/s3_ops.py`:

```python
"""S3 operations via AWS CLI subprocess.
Consistent with deploy_static_site.py — no boto3.
"""
import io
import json
import subprocess
import tempfile
from pathlib import Path


def s3_get_bytes(bucket: str, key: str) -> bytes | None:
    """
    Download a file from S3. Returns bytes or None if not found.
    Uses: aws s3 cp s3://{bucket}/{key} -
    """
    result = subprocess.run(
        ["aws", "s3", "cp", f"s3://{bucket}/{key}", "-"],
        capture_output=True,
    )
    if result.returncode == 0:
        return result.stdout
    return None   # key does not exist or access denied


def s3_put_bytes(bucket: str, key: str, data: bytes,
                 content_type: str, cache_control: str,
                 acl: str = "public-read") -> bool:
    """
    Upload bytes to S3.
    Uses a temp file because aws s3 cp does not support piped stdin reliably.
    """
    with tempfile.NamedTemporaryFile(delete=False, suffix=Path(key).suffix) as f:
        f.write(data)
        tmp_path = f.name

    result = subprocess.run(
        [
            "aws", "s3", "cp", tmp_path, f"s3://{bucket}/{key}",
            "--content-type",  content_type,
            "--cache-control", cache_control,
            "--acl",           acl,
        ],
        capture_output=True, text=True,
    )
    Path(tmp_path).unlink(missing_ok=True)

    if result.returncode != 0:
        print(f"  ERROR uploading s3://{bucket}/{key}: {result.stderr}")
        return False
    return True


def s3_put_json(bucket: str, key: str, data: dict,
                cache_control: str = "public, max-age=60") -> bool:
    """Upload a Python dict as JSON to S3."""
    return s3_put_bytes(
        bucket        = bucket,
        key           = key,
        data          = json.dumps(data, indent=2).encode("utf-8"),
        content_type  = "application/json",
        cache_control = cache_control,
    )


def s3_get_json(bucket: str, key: str) -> dict | None:
    """Download and parse JSON from S3. Returns None if not found."""
    raw = s3_get_bytes(bucket, key)
    if raw is None:
        return None
    return json.loads(raw.decode("utf-8"))
```

### Locale matrix

Run browser tests in a matrix across the four Phase 1 locales. Each matrix job captures
screenshots for one locale and contributes to the shared manifest.

```yaml
strategy:
  matrix:
    locale: [en-gb, pt-pt, de-de, fr-fr]
  fail-fast: false    # don't cancel other locales if one fails
```

**Manifest write coordination:** Because each locale runs as a separate CI job, each job
writes to the manifest independently. The manifest is read at the start of each job and
written at the end. This creates a potential race condition if two locale jobs finish
simultaneously.

This is slower but correct. A proper distributed locking solution (e.g., S3 conditional
writes) can be added in Phase 2 when test volume justifies parallel runs.

## Part 10 — The QA Site: How It Renders Screenshots

The QA site HTML does not embed screenshots. It references them via S3 URLs resolved
through the manifest. Here is how a use-case page renders at runtime.

### The manifest is fetched client-side

Each use-case page includes a `<script>` that fetches the manifest from S3 and renders
screenshot `<img>` tags dynamically:

```html
<!-- en-gb/use-cases/sgraph-ai/pricing/index.html -->
<div id="screenshots-container" 
     data-target="sgraph-ai"
     data-locale="pt-pt"
     data-page="pricing"
     data-steps='["01_full_page","02_above_fold","03_nav"]'>
  <p>Loading screenshots...</p>
</div>

<script>
const MANIFEST_URL  = "https://745506449035--static-sgraph-ai--eu-west-2.s3.eu-west-2.amazonaws.com/websites/sgraph-qa/screenshots/sgraph-ai/manifest.json";
const BUCKET_BASE   = "https://745506449035--static-sgraph-ai--eu-west-2.s3.eu-west-2.amazonaws.com";
const S3_PREFIX     = "websites/sgraph-qa/screenshots";

async function loadScreenshots() {
    const container = document.getElementById("screenshots-container");
    const target    = container.dataset.target;
    const locale    = container.dataset.locale;
    const page      = container.dataset.page;
    const steps     = JSON.parse(container.dataset.steps);

    const resp     = await fetch(MANIFEST_URL);
    const manifest = await resp.json();

    container.innerHTML = "";

    for (const step of steps) {
        const entryKey = `${locale}/${page}/${step}`;
        const version  = manifest.entries[entryKey];

        if (!version) {
            container.innerHTML += `<p class="no-screenshot">No screenshot for ${entryKey}</p>`;
            continue;
        }

        const imgUrl      = `${BUCKET_BASE}/${S3_PREFIX}/${target}/${version}/${entryKey}.png`;
        const analysisUrl = `${BUCKET_BASE}/${S3_PREFIX}/${target}/${version}/${entryKey}.json`;

        // Screenshot
        container.innerHTML += `
          <div class="screenshot-block">
            <h3>${step}</h3>
            <p class="version-badge">captured at ${version}</p>
            <img src="${imgUrl}" alt="${step} — ${locale}" loading="lazy">
            <div class="analysis-block" data-analysis-url="${analysisUrl}">
              <!-- LLM analysis renders here when hook is wired -->
              <span class="analysis-pending">Analysis pending</span>
            </div>
          </div>`;
    }
}

loadScreenshots();
</script>
```

### What the "version badge" communicates

The `<p class="version-badge">captured at v0.12.3</p>` is meaningful to the user:
it tells them *when this visual state was introduced*. If the current target version is
v0.13.8 and the screenshot badge says v0.12.3, that page has not changed visually in
three target releases. If it says v0.13.8, it changed in the most recent release.

This is the visual changelog made human-readable.
