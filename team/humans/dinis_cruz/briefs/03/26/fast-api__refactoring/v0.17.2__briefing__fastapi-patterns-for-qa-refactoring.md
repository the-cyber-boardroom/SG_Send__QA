**Purpose:** Everything you need to know about how SG/Send uses `osbot-fast-api`, `osbot-fast-api-serverless`, and `fastapi-mcp` — so you can bring the QA API (`sg_send_qa/api/`) into alignment.

## 1. The Three Packages

| Package | Role | QA Repo Status |
|---------|------|---------------|
| `osbot-fast-api` | Route classes (`Fast_API__Routes`), `route_path` decorator, `Fast_API_Server` (test HTTP server), service registry, client requests | **Installed** (transitive) — only `Fast_API_Server` is used (test harness) |
| `osbot-fast-api-serverless` | `Serverless__Fast_API` base class (Lambda + Mangum), `Routes__Info`, deploy helpers | **Installed** — not used by the QA API at all |
| `fastapi-mcp` | `FastApiMCP` for mounting MCP servers | **Installed** — not used by the QA API yet |

The QA API currently uses vanilla `FastAPI()` + `APIRouter`. The goal is to bring it onto the osbot patterns.

## 2. The Core Pattern: `Serverless__Fast_API` Subclass

Every FastAPI app in the main repo extends `Serverless__Fast_API` and implements three lifecycle methods. Here is the pattern you need to follow:

```python
from osbot_fast_api_serverless.fast_api.Serverless__Fast_API import Serverless__Fast_API

class Fast_API__QA__Server(Serverless__Fast_API):

    # Declare services as Type_Safe attributes
    qa_runner : QA_API__Runner = None

    def setup(self):
        with self.config as _:
            _.name        = 'SG/Send QA API'
            _.version     = version__sg_send__qa
            _.description = 'Browser automation API for SG/Send QA'
            _.enable_api_key = False                    # or True for auth-gated

        # Wire services (using the if-None guard pattern for test injection)
        if self.qa_runner is None:
            self.qa_runner = QA_API__Runner()

        return super().setup()

    def setup_routes(self):
        self.add_routes(Routes__Info)                   # standard /info/health etc.
        self.add_routes(Routes__Browser,
                        qa_runner = self.qa_runner)
        self.add_routes(Routes__Upload,
                        qa_runner = self.qa_runner)
        # ... etc
```

### Lifecycle

1. **`setup()`** — configure `self.config`, create service instances, call `super().setup()`
2. **`setup_routes()`** — register route classes via `self.add_routes(RouteClass, **kwargs)`
3. **`handler()`** (inherited) — returns Mangum handler for Lambda. For local dev, use `.app()` for the ASGI app or `.client()` for a TestClient.

### Lambda Handler (if/when QA goes serverless)

```python
with Fast_API__QA__Server() as _:
    _.setup()
    handler = _.handler()
    app     = _.app()

def run(event, context=None):
    return handler(event, context)
```

### Test Client

```python
fast_api = Fast_API__QA__Server().setup()
client   = fast_api.client()         # Starlette TestClient — in-process, no HTTP
response = client.post('/api/browser/open', json={...})
```

## 3. The Route Class Pattern: `Fast_API__Routes`

Replace `APIRouter` with `Fast_API__Routes` subclasses. Each domain gets its own class.

### Current QA pattern (to be replaced)

```python
router = APIRouter(prefix="/api/browser", tags=["browser"])

@router.post("/open")
def browser_open(body: dict = Body(default={})):
    req = Schema__QA_Request(**{k: v for k, v in body.items() ...})
    ...
```

### Target pattern

```python
from osbot_fast_api.api.routes.Fast_API__Routes import Fast_API__Routes

class Routes__Browser(Fast_API__Routes):
    tag        : str = 'browser'
    qa_runner  : QA_API__Runner                  # injected via add_routes()

    def open(self, body: Schema__QA_Request) -> dict:   # POST /browser/open
        def workflow(session):
            session.sg_send.page__root()
            return {"url": session.sg_send.url(), "title": session.sg_send.title()}
        return self.qa_runner.run_with_request(body, workflow)

    def screenshot(self, body: Schema__QA_Request) -> dict:   # POST /browser/screenshot
        ...

    def setup_routes(self):
        self.add_route_post(self.open)
        self.add_route_post(self.screenshot)
        return self
```

### How method names map to paths

The `Fast_API__Routes` convention is:
- Method `open` → `POST /{tag}/open` → `/browser/open`
- Method `lookup__token_name` → `GET /{tag}/lookup/{token_name}`
- Double underscores become path parameters

For complex paths (wildcards, colons), use the `@route_path` decorator:

```python
from osbot_fast_api.api.decorators.route_path import route_path

@route_path('/custom/{some_id:path}')
def custom__some_id(self, some_id: str, request: Request) -> dict:
    ...
```

## 4. Service Injection

Routes receive dependencies through `Type_Safe` constructor kwargs, injected at `add_routes()` time in the app's `setup_routes()`:

```python
# In the app:
self.add_routes(Routes__Browser,
                qa_runner = self.qa_runner)

# In the route class:
class Routes__Browser(Fast_API__Routes):
    qa_runner: QA_API__Runner           # Type_Safe sets this from the kwargs
```

This is **not** FastAPI's `Depends()` system. The entire service graph is wired once at startup.

### The `if-None` guard pattern

Every service in `setup()` uses this pattern so tests can inject alternatives:

```python
if self.qa_runner is None:
    self.qa_runner = QA_API__Runner()
```

This means a test can do:

```python
custom_runner = QA_API__Runner(some_config=...)
app = Fast_API__QA__Server(qa_runner=custom_runner).setup()
```

No mocks. No patches. Just constructor injection via Type_Safe.

## 5. Schema Pattern: `Type_Safe` (Never Pydantic)

The QA repo already follows this correctly — `Schema__QA_Request`, `Schema__QA_Response`, and `Schema__Capture_Config` all extend `Type_Safe`. Keep this.

What needs to change: the route handlers should accept the `Type_Safe` schema directly as the `body` parameter, not `dict = Body(default={})`. FastAPI will deserialise the JSON body into the schema.

```python
# Current (wrong):
def browser_open(body: dict = Body(default={})):
    req = Schema__QA_Request(**{k: v for k, v in body.items() ...})

# Target (correct):
def open(self, body: Schema__QA_Request) -> dict:
    # body is already a Schema__QA_Request
```

## 6. The Two QA FastAPI Apps — Merge Them

The QA repo currently has two disconnected FastAPI applications:

| App | File | Endpoints | Purpose |
|-----|------|-----------|---------|
| QA API | `sg_send_qa/api/QA_API__Server.py` | `/api/browser/*`, `/api/upload/*`, `/api/download/*`, `/api/workflow/*`, `/api/qa/health` | Browser automation |
| Server | `sg_send_qa/server/main.py` | `/info/health`, `/api/tests/run`, `/api/docs`, `/site/*` | Test runner + docs |

Both should become one `Serverless__Fast_API` subclass with all routes registered via `add_routes()`. The static site mount goes in `setup_routes()` just as the main repo mounts admin UI statics.

## 7. Inter-Lambda Communication (for reference)

The main repo's User Lambda calls Admin Lambda for token validation using the osbot service registry:

```
Admin__Service__Client → Admin__Service__Client__Requests (extends Fast_API__Client__Requests)
  → Service Registry → HTTP to Admin Lambda URL
```

The QA API doesn't need inter-Lambda calls today, but if it later needs to call the SG/Send Admin Lambda directly (e.g. to create tokens programmatically), this is the pattern.

## 8. MCP Integration (for reference)

The main repo mounts MCP servers via the `MCP__Setup` utility class:

```python
mcp_setup = MCP__Setup(name='sgraph-send-user',
                       include_tags=['api/transfers', 'api/vault'],
                       stateless=True)
self.mcp = mcp_setup.mount_mcp(self.app())
```

The QA API could expose MCP tools to let Claude trigger test runs, capture screenshots, or extract page models. This is future work but the infrastructure is ready — the QA repo already depends on `fastapi-mcp`.

## 9. Auth Configuration

`Serverless__Fast_API` provides `enable_api_key` in config:

| Setting | Effect |
|---------|--------|
| `True` (default) | Global API key middleware — requires header matching env var |
| `False` | No global auth — each route handles its own auth |

For the QA API, `enable_api_key = False` is probably correct initially (the API is internal tooling). Per-route auth can be added later if the API is exposed.

## 10. Test Infrastructure

The main repo's test pattern that should be replicated:

```python
from osbot_utils.type_safe.Type_Safe import Type_Safe

class Fast_API__Test_Objs__QA(Type_Safe):
    fast_api        : Fast_API__QA__Server = None
    fast_api__app   : FastAPI              = None
    fast_api__client: TestClient           = None

_test_singleton = None

def setup__qa_fast_api_test_objs():
    global _test_singleton
    if _test_singleton is None:
        fast_api = Fast_API__QA__Server().setup()
        _test_singleton = Fast_API__Test_Objs__QA(
            fast_api         = fast_api,
            fast_api__app    = fast_api.app(),
            fast_api__client = fast_api.client(),
        )
    return _test_singleton
```

Tests then use the singleton's `fast_api__client` to make requests. No mocks. No patches. The test harness can inject a stub session class via the `if-None` guard to avoid starting Playwright for unit tests.

## 11. Summary: What to Refactor

| Current | Target |
|---------|--------|
| `FastAPI()` in `QA_API__Server.py` | `Serverless__Fast_API` subclass |
| `FastAPI()` in `server/main.py` | Merge into the same subclass |
| `APIRouter` per route file | `Fast_API__Routes` subclass per domain |
| `dict = Body(default={})` | `Schema__QA_Request` body parameter |
| `_req(body)` helper in every route | Automatic — FastAPI deserialises to Type_Safe |
| `unittest.mock.patch` in tests | Type_Safe constructor injection |
| Hardcoded version string | Read from `sg_send_qa/version` |
| No `/info/health` | `Routes__Info` added automatically |
