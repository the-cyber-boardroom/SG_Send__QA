### Background

- Playwright + Chromium are **not** pre-installed in Claude cloud tasks — the prompt installs them each run (~60s)
- Python default is 3.11; a 3.12 venv is created at `/tmp/venv312` (see `v0.2.10__playwright-setup-guide.md`)
- Shell state does not persist between tool calls — every command prefixes `source /tmp/venv312/bin/activate &&`
- **Proxy + TLS interception** — the Claude cloud environment routes all traffic through an
  authenticated egress proxy (`21.0.0.25:15004`). Playwright does **not** pick up `https_proxy`
  env vars automatically; the proxy must be passed explicitly to `p.chromium.launch(proxy=...)`.
  The proxy also performs TLS interception, so `ignore_https_errors=True` is required on the
  browser context. Both fixes are in the suite scripts.
  `curl` works fine because it reads the env vars natively — if curl succeeds but Playwright
  returns HTTP 407, the missing explicit proxy config is the cause.
- **Suite scripts** live in `tests/scheduled/` — edit them there, not in this prompt.

## Run All Suites

Suite scripts live in `tests/scheduled/`. Any file matching `suite_*.py` is
picked up automatically in filename order — add a new suite file and it runs
next time without changing this prompt.

source /tmp/venv312/bin/activate && python - <<'RUNNER'
import subprocess, sys, os, glob

scripts = sorted(glob.glob("tests/scheduled/suite_*.py"))
if not scripts:
    print("No suite scripts found in tests/scheduled/")
    sys.exit(1)

overall = True
for script in scripts:
    print(f"\n{'='*60}")
    print(f"Running: {script}")
    print('='*60)
    result = subprocess.run([sys.executable, script])
    if result.returncode != 0:
        overall = False

sys.exit(0 if overall else 1)
RUNNER
SUITES_EXIT=$?
echo "All suites exit: $SUITES_EXIT"
